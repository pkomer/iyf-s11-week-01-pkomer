Why Semantic HTML and Accessibility Matter

Semantic HTML means using HTML tags that clearly describe their purpose. Instead of using many <div> tags, we can use tags like <header>, <nav>, <main>, and <footer>. These tags help browsers, search engines, and screen readers understand the structure of a webpage

Before: Non-Semantic HTML

Before semantic HTML was introduced, web developers commonly used generic elements such as <div> and <span> to build web pages. While these elements helped structure content, they did not describe the purpose of the content they contained. As a result, it was more difficult for search engines, screen readers, and other tools to understand the layout and meaning of a webpage. Developers often relied on class names and IDs to indicate the role of different sections.

After: Semantic HTML

After the introduction of semantic HTML, developers gained access to meaningful elements such as <header>, <nav>, <main>, <section>, <article>, and <footer>. These elements clearly describe the purpose of the content they contain, making web pages easier to read, maintain, and understand. Semantic HTML improves accessibility for users with assistive technologies and helps search engines better interpret and index website content, leading to a more organized and user-friendly web experience.

Missing Image Descriptions

Some images on the webpage were missing alternative text (alt attributes). Alternative text is important because it provides a description of an image for users who cannot see it, including those who rely on screen readers. Without alt text, screen readers may be unable to communicate the purpose or content of an image, reducing the accessibility of the website. Adding clear and descriptive alt text helps ensure that all users can understand the information conveyed by images and improves the overall user experience and accessibility of the webpage

Poor Heading Structure

There were also some issues with headings being out of order. This may be confusing for people who use screen readers. I fixed this by using headings in the correct order, such as <h1>, <h2>, and <h3>.

Keyboard Navigation

Some elements on the website were not accessible through the keyboard alone. I ensured that all the links and buttons were accessible by using the Tab key. All these improvements increased the accessibility of my website, making it more user-friendly. Semantic HTML and accessibility are important because they allow us to build websites that are understandable, easy to maintain, and accessible to everyone. In all future projects, accessibility will be an integral part of my web development journey
